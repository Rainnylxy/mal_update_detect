Вернемся к созданию клиента вводим айпи и порт из плейита, это важно, чтобы все работало... Ждем компиляцию и все можно использовать.

Greetings, my friend!

To start using the rat, you need to perform a couple of simple actions to get everything working - 

1. Download Python 3.13 and Playit to open the ports so you can use it remotely instead of locally. 

2. Open the terminal in the folder with the program and then enter the following commands.
	Set up a virtual environment - python -m venv venv
	Activate it - venv/scripts/activate
	Install the dependencies - pip install -r requirements.txt
	Wait for the installation and you can run the file run.bat.

3. Creating a client - go to the build tab in the interface and enter the play's IP address, you must first create 2 tunnels for the ports that rat uses, namely: port 5000 and port 6000 - these are the default ports, you can specify your own ports in the Settings tab, and if you change them, set the IP to localhost.

Let's return to creating the client by entering the IP and port from the play, it's important for everything to work... We wait for the compilation and everything can be

I emphasized that no matter what bug it is, you did not lose the user.Also, at this moment, you need to know - if something went wrong or something is not working, just restart the rat.
Do not delete the settings.json file, otherwise the users will lose access, as there is a unique key that grants access to the backend.
